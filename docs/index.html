Product modules

A notebook interface designed like an operations workspace.

The design follows a compact admin-workbench pattern: persistent navigation, dense cards, direct actions, explicit metadata, and status panels that remain useful on a laptop screen.

01

Import and storage

Upload HTML into generated or imported libraries while keeping raw content outside Git.

02

Browse and filter

Use collections, tags, favorites, archive state, search, sorting, and tag AND/OR filters.

03

Reader workflow

Open full HTML notes, copy links, share entries, edit metadata, favorite, archive, or restore.

04

AI-ready sidebar

Prepare contextual Q&A and note generation with manually selected context files.

05

Settings and data

Scaffold provider settings, data backup, WebDAV, export, theme, language, and account modules.

06

Deployable app

Run as a PWA with a static frontend and optional API backend in one Docker service.

Reusable deployment

One project, two modes: static preview and Docker notebook.

GitHub Pages can host the public intro and read-only demo. Docker runs the real notebook with the backend API, upload handling, metadata persistence, archive delete protection, and rebuild flow.

Docker Pages PWA 
git clone https://github.com/JMoCoder/html_vault.git
cd html_vault
docker compose up -d --build

# open http://localhost:8080
